## 项目结构

```
/
├── app/                 # Next.js App Router
│   ├── layout.tsx       # 根布局
│   ├── page.tsx         # 首页
│   └── docs/            # 文档路由
├── components/          # 共享组件
│   ├── navbar.tsx       # 顶部导航栏
│   ├── sidebar.tsx      # 侧边栏
│   └── toc.tsx          # 目录
├── content/             # MDX 文档内容
│   ├── getting-started.mdx
│   └── dev/
├── lib/
│   └── source.ts        # 侧边栏配置
├── tailwind.config.ts   # Tailwind 配置
├── next.config.ts       # Next.js 配置
├── package.json
└── README.md
```

## 本地开发

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

Next.js 会在 `http://localhost:3000` 启动热重载开发服务器。

### 构建生产版本

```bash
npm run build
```

构建产物输出到 `out/` 目录，为纯静态文件。

---

## 添加新文档

1. 在 `content/` 目录创建 `.mdx` 文件，例如：`content/tutorial.mdx`
2. 在 `app/docs/` 下创建路由目录并添加 `page.tsx` 导入该 MDX 文件
3. 在 `lib/source.ts` 的 `sidebar` 中添加条目
4. 访问 `/docs/tutorial` 查看

---

## 部署

项目输出纯静态文件，可一键部署到：

- **Vercel**: Framework Preset: `Next.js`，自动检测
- **Netlify**: Build Command `npm run build`, Publish Directory `out`
- **Cloudflare Pages**: Build Command `npm run build`, Output Directory `out`
