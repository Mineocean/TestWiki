## 技术栈

- **框架**: Next.js 15（App Router）
- **语言**: TypeScript + MDX
- **样式**: Tailwind CSS v3
- **内容**: MDX（Markdown + React 组件）
- **部署**: Vercel / Netlify / Cloudflare Pages 等静态托管平台

## 整体架构

```
┌─────────────────────────────────────┐
│           浏览器 / 用户              │
└─────────────┬───────────────────────┘
              │ HTTPS
┌─────────────▼───────────────────────┐
│        静态托管平台                   │
│   ┌─────────────────────────────┐   │
│   │   out/ (静态输出)            │   │
│   │   ├── index.html            │   │
│   │   ├── docs/                 │   │
│   │   │   ├── getting-started.html  │
│   │   │   └── dev/              │   │
│   │   └── _next/static/          │   │
│   └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

## 构建流程

1. **源文件**: `.mdx` 文件存放在 `content/` 目录
2. **路由映射**: `app/docs/` 下的 `page.tsx` 文件导入对应 MDX 内容
3. **MDX 编译**: `@next/mdx` 在构建时将其编译为 React 组件
4. **样式处理**: Tailwind CSS 生成优化的 CSS，`@tailwindcss/typography` 处理文档排版
5. **静态导出**: `next build` → `out/` 目录，纯静态 HTML/CSS/JS

## 目录约定

| 目录/文件 | 用途 |
|-----------|------|
| `app/layout.tsx` | 根布局（字体、metadata） |
| `app/page.tsx` | 首页 |
| `app/docs/layout.tsx` | 文档布局（导航栏 + 侧边栏 + 目录） |
| `app/docs/*/page.tsx` | 各文档路由（导入 MDX 内容） |
| `components/` | 可复用 UI 组件 |
| `content/` | MDX 文档源文件 |
| `lib/source.ts` | 侧边栏配置 |
| `mdx-components.tsx` | 全局 MDX 自定义组件 |

## 路由映射

Next.js App Router 文件路由：

- `app/page.tsx` → `/`
- `app/docs/getting-started/page.tsx` → `/docs/getting-started`
- `app/docs/dev/architecture/page.tsx` → `/docs/dev/architecture`
- `app/docs/dev/api-reference/page.tsx` → `/docs/dev/api-reference`

## 扩展点

- **自定义组件**: 在 `mdx-components.tsx` 中定义 MDX 可用组件
- **布局定制**: 修改 `app/docs/layout.tsx` 调整文档页面布局
- **插件**: 支持 Remark / Rehype 插件生态系统
