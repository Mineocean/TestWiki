> Next.js 静态导出站点没有运行时 API。以下说明涵盖站点的 URL 结构和配置接口。

---

## URL 结构

| 路径 | 说明 |
|------|------|
| `/` | 首页（`app/page.tsx`） |
| `/docs/getting-started` | 快速开始指南 |
| `/docs/dev/architecture` | 架构设计文档 |
| `/docs/dev/api-reference` | 接口说明（本页） |

---

## 侧边栏配置

侧边栏在 `lib/source.ts` 中定义：

```typescript
export interface SidebarGroup {
  text: string
  items: { text: string; link: string }[]
}

export const sidebar: SidebarGroup[] = [
  {
    text: '起步',
    items: [
      { text: '快速开始', link: '/docs/getting-started' },
    ],
  },
  {
    text: '开发指南',
    items: [
      { text: '架构设计', link: '/docs/dev/architecture' },
      { text: '接口说明', link: '/docs/dev/api-reference' },
    ],
  },
]
```

## 添加新文档

1. 在 `content/` 目录创建 `.mdx` 文件
2. 在 `app/docs/` 下创建对应路由目录和 `page.tsx`
3. 示例：

```tsx
// app/docs/tutorial/page.tsx
import type { Metadata } from 'next'
import Content from '@/content/tutorial.mdx'

export const metadata: Metadata = {
  title: '教程 | Markdown Wiki',
}

export default function Page() {
  return <Content />
}
```

4. 在 `lib/source.ts` 的 `sidebar` 中添加条目

---

## 全局 MDX 组件

`mdx-components.tsx` 位于项目根目录，`@next/mdx` 自动识别并应用于所有 MDX 文件：

- 链接（`a`）自动使用 Next.js `Link` 组件
- 代码块（`pre`）应用圆角边框和背景
- 表格（`table`/`th`/`td`）优化排版
- 引用块（`blockquote`）使用紫色左边框
